Voice Call Flow

What happens from the moment a tenant dials the Twilio number to ticket creation, emergency escalation, and beyond.

📞

Tenant Calls Twilio

Tenant dials the Twilio number +1 (207) 406-9385. Twilio receives the call and fires a webhook to your backend with the caller's phone number.

// Twilio fires webhook POST /api/twilio/voice Form data: { From: "+12075551234", To: "+12074069385" }
⚙

handle_incoming_call() twilio_webhooks.py:46

Backend checks if ElevenLabs is configured. If the agent ID is missing or set to "placeholder", the caller gets a fallback message.

✕ Not configured

Caller hears: "Our automated system is currently being set up. Please text this number with your maintenance request."

✓ Configured

Proceeds to register the call with ElevenLabs API.

☁

ElevenLabs Register-Call ElevenLabs Backend

Backend makes an outbound API call to ElevenLabs to register this phone call. ElevenLabs returns TwiML that tells Twilio to connect the audio stream to their WebRTC servers.

POST https://api.elevenlabs.io/v1/convai/twilio/register-call { "agent_id": "agent_6101kjt3qgzpfd1vq3kwgm1a68d4", "from_number": "+12075551234", "to_number": "+12074069385", "direction": "inbound" } // Returns TwiML → Twilio connects audio to ElevenLabs

✕ API fails

Caller hears: "We are experiencing a temporary issue. Please text this number..."

✓ Success

Caller is now live-connected to the ElevenLabs voice AI agent.

🗣

Voice AI Conversation ElevenLabs Agent

The ElevenLabs agent (configured in their dashboard) speaks to the tenant in natural language. It gathers:

Agent gathers: - Tenant name - Property name (can verify via lookup_property tool) - Unit number - Description of the issue - Urgency assessment The agent has 3 server tools it can call back to your API: 1. POST /api/tools/lookup_property 2. POST /api/tools/create_ticket 3. POST /api/tools/escalate_emergency
🔍

Tool: lookup_property elevenlabs_tools.py:89 Database

Agent verifies the property exists in your system. Case-insensitive search. If found, agent can confirm address with the caller.

POST /api/tools/lookup_property { "property_name": "Oak Apartments" } // DB: Property.name ILIKE '%Oak Apartments%' Found → { found: true, name, address, manager_name, manager_phone } Not found → { found: false, message: "Property not found" }
🎫

Tool: create_ticket elevenlabs_tools.py:43 Database

When the agent has enough info, it calls create_ticket. This creates a Ticket record with channel = "phone" and auto-assesses urgency if not provided.

POST /api/tools/create_ticket { "tenant_name": "John Smith", "tenant_phone": "+12075551234", "unit_number": "3B", "property_name": "Oak Apartments", "description": "Water dripping from ceiling in bathroom", "category": "plumbing", "urgency": "high" } // → create_ticket_from_conversation() // → Validates enums, auto-assesses urgency if missing // → Creates Ticket(status=open, channel=phone)
⚠

Urgency Auto-Assessment ticket_service.py:26

If the ElevenLabs agent doesn't set urgency (or sends an invalid value), the backend scans the description for keywords. Maine-specific: HVAC issues default to HIGH.

EMERGENCY keywords: "no heat", "burst pipe", "flooding", "gas leak", "fire", "smoke", "carbon monoxide", "sewage", "roof collapse", "exposed wire" HIGH keywords: "no hot water", "major leak", "power out", "fridge not working", "outlet sparking" Special rule: category == HVAC → at least HIGH (Maine winters)

🚀

What Happens Next

The ticket is now in the system. From here, the rest of the pipeline kicks in:

1. Dashboard — Ticket appears as "open" in web UI Manager sees it in ticket list + stats 2. Assignment Agent — Manager chats with AI "Assign open tickets" → AI matches skills → Creates Assignment with AM/PM time block → SMS sent to employee with job details 3. Follow-Up — Scheduler fires at noon/5pm → SMS to employee: "Did you finish?" → Employee replies with photos (MMS) 4. Feedback — Tenant gets satisfaction survey → Claude Haiku analyzes: score 1-5 → Ticket marked RESOLVED

Current Gaps & Improvement Opportunities

1
No caller phone capture guarantee — The caller's phone number is passed to ElevenLabs via register-call, but the voice agent must explicitly pass it back as tenant_phone in the create_ticket tool call. If the agent doesn't, the ticket has no phone for follow-up SMS.
2
No conversation transcript — Unlike SMS (saved to Conversation model), voice calls don't persist what was said. There's no record of the AI-tenant conversation.
3
No dropped call handling — The /voice/status endpoint just logs. Could detect "failed" or "no-answer" and trigger a follow-up SMS to the caller.
4
ElevenLabs config is external — The agent's personality, prompt, and tool schemas live in the ElevenLabs dashboard, not in code. If misconfigured, it won't gather the right info or call the right tools.
