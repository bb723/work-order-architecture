How Each Channel Works

Three intake channels, each wired to different AI and communication services.

SMS Intake

services/sms_agent.py → process_sms_message()

Tenant texts in. Twilio delivers to POST /api/twilio/sms. The SMS router classifies the sender, then Claude Sonnet runs a multi-turn conversation to gather name, unit, property, and issue details.

  1. 1handle_incoming_sms() receives Twilio form data
  2. 2classify_sender() routes to intake, completion, or feedback
  3. 3Claude Sonnet chats with tenant using 2 tools: create_ticket and escalate_emergency
  4. 4create_ticket_from_conversation() persists the Ticket to SQLite
  5. 5If emergency → check_and_escalate() triggers outbound call

Voice Intake

routers/twilio_webhooks.py → handle_incoming_call()

Tenant calls. Twilio hits POST /api/twilio/voice, which calls the ElevenLabs register-call API to connect the caller to a voice AI agent. The agent calls backend tool endpoints to create tickets.

  1. 1Twilio voice webhook fires, backend calls ElevenLabs API
  2. 2ElevenLabs returns TwiML to connect WebRTC stream
  3. 3Voice agent calls POST /api/tools/create_ticket
  4. 4Can also call /api/tools/lookup_property and /api/tools/escalate_emergency

AI Assignment Agent

services/assignment_agent.py → process_assignment_chat()

Manager chats with Claude via POST /api/agent/chat. The agent runs up to 10 tool-calling rounds using 7 tools to check skills, workloads, schedules, and make assignments.

  1. 1AgentSession tracks conversation in DB (JSON messages)
  2. 2Claude calls get_unassigned_tickets + get_employees
  3. 3Matches skills via SkillAssessment (9 categories, 4 levels)
  4. 4assign_ticket creates Assignment + sends SMS to employee
  5. 5Manager corrections → update_employee_skill + AssignmentFeedback audit

Completion & Feedback

services/completion_handler.py + followup_scheduler.py

APScheduler cron jobs fire at noon and 5 PM to send follow-up texts. Employees reply with MMS photos. Tenants get a satisfaction survey analyzed by Claude Haiku.

  1. 1trigger_am_followups() at 12:00 PM / trigger_pm_followups() at 5:00 PM
  2. 2Employee texts back → handle_employee_completion() extracts MMS photos
  3. 3send_feedback_request_to_tenant() surveys the tenant
  4. 4analyze_tenant_feedback() uses Claude Haiku → score 1-5, resolved flag, summary

Emergency Escalation

services/emergency.py → trigger_emergency_call()

When AI detects an emergency (burst pipe, no heat, gas leak), check_and_escalate() places an outbound Twilio call to the property manager with a TwiML voice announcement.

  1. 1AI sets urgency = "emergency" during intake
  2. 2trigger_emergency_call(ticket) builds TwiML Say announcement
  3. 3Twilio calls EMERGENCY_PHONE_NUMBER with details
  4. 4Ticket marked emergency_dispatched = True

Management Dashboard

frontend/src/pages/ (React + Vite + Tailwind)

Five pages give managers full visibility: ticket dashboard with live stats, employee skill management, weekly/daily schedule, AI chat interface, and property settings.

  1. 1Dashboard.jsx — stats cards, filters, auto-refresh every 15s
  2. 2Employees.jsx — skill grid (9 categories × 4 levels, color-coded)
  3. 3Schedule.jsx — AM/PM time blocks, weekly or daily toggle
  4. 4AgentChat.jsx — chat UI with quick prompts, shows tool actions
